# 支付宝二维码收款配置

## 当前已配置

网页已经改成：

- 会员价格：`299 元/月`
- 收款方式：支付宝收款二维码
- 开通方式：人工核对到账后发放开通口令
- 会员有效期：30 天
- 付费后解锁：`9:40 可买入个股` 模块

## 一、支付宝收款码已上线

当前二维码已替换为你的真实支付宝收款码：

```text
assets/payment/alipay-qr.jpg
```

`limit-up-panel.html` 中的引用为：

```html
<img class="pay-qr" src="./assets/payment/alipay-qr.jpg" alt="支付宝收款二维码 · 299元包月">
```

如果以后要更换收款码，直接覆盖同名 `assets/payment/alipay-qr.jpg` 文件即可，无需改 HTML。

## 二、默认开通口令

当前默认开通口令是：

```text
ZT940-299
```

用户支付后，你把这个口令发给用户。用户输入后，页面会在本机浏览器保存 30 天会员状态。

## 三、如何更换开通口令

建议你定期更换口令，例如每月一个：

```text
ZT940-202607
ZT940-202608
```

生成新口令哈希：

```bash
python - <<'PY'
import hashlib
code = "ZT940-202607"
print(hashlib.sha256(code.encode()).hexdigest())
PY
```

然后打开：

```text
assets/member-paywall.js
```

替换这里的哈希：

```js
var ACTIVATION_CODE_HASHES = [
  "你的新口令哈希"
];
```

如果你想让多个口令同时有效，可以写多个哈希：

```js
var ACTIVATION_CODE_HASHES = [
  "第一个口令哈希",
  "第二个口令哈希",
  "第三个口令哈希"
];
```

## 四、推荐人工收款流程

1. 用户打开网页，点击 `会员登录 / 开通`。
2. 用户扫码支付 `299 元`。
3. 用户付款备注填写手机号或微信号。
4. 你在支付宝账单里核对到账。
5. 你把当月开通口令发给用户。
6. 用户输入口令后，解锁 9:40 模块 30 天。

## 五、必须知道的限制

这个方案不接支付 API，不需要服务器，因此免费、简单，但安全性有限：

- 技术用户仍可能通过浏览器绕过前端门禁。
- 不能自动核验支付宝是否到账。
- 不能自动续费。
- 不能统一管理会员。
- 用户换浏览器或清缓存后，本机会员状态会丢失，需要重新输入口令。

如果开始真实收费并有较多用户，建议升级为：

```text
Cloudflare Pages
+ Cloudflare Workers
+ D1 数据库
+ 支付宝当面付 / 微信支付 Native
+ 服务端会员鉴权
```
